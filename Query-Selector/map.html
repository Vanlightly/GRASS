  • Advanced Queries
  • Components Without a Given Relationship

With Cypher, it is not obvious how to match nodes WITHOUT a given relationship.

  1. Collect the names of components (of the desired type) that you do NOT want returned.
  2. Match ALL components of the desired type
  3. Filter components not in the collected list of components

The example below shows all applications that do not access databases

match (a:Application)-->(b:Database)
with collect(a.name) as appsThatAccessDbs
match (n:Application)
with n.name as appName
where not appName in appsThatAccessDbs
return appName
order by appName

A direct or multi-hop circular dependency is poor design.

In the interests of query execution time, the below query limits the hops to 5

match(aStart)-[:REFERENCES*1..5]->(aFinish)
where aStart.name = aFinish.name
return aStart, aFinish
